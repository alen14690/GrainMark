## 🔍 方案合理性复盘（专家自我 Review）

### 👍 好的设计决策

1. **SecurityError.code 作为 API 契约**：message 给人读，code 给测试断言 — 这让测试稳定
2. **Zod IPC Schema 与 shared/ 共享**：主/渲染统一契约，未来接入类型化 ipc 客户端更容易
3. **PathGuard 在 Main 级初始化 + Dialog 自动 addAllowed**：用户每次选择新目录就扩权，体验流畅且安全
4. **Logger 自动脱敏**：不靠开发者自觉，架构级保证
5. **不依赖 better-sqlite3**：JsonTable/KV 足够应对 P1，规避了本机 xcode CLT 问题，接口保持可平滑切换

### 🤔 可能的更好方案（未来 Pass 可重审）

1. **是否该用 better-sqlite3 打包 prebuilt binary？**
   - 方案 A：`electron-rebuild` + prebuilt binary CI 发布 — 长期更好
   - 方案 B（当前）：JSON 存储 — 开发快，数据量小时无性能差
   - **结论**：M1.5 保持 JSON；到 M2 数据量增大前再切换

2. **grain:// 协议 vs `net.fetch(pathToFileURL)` 直通**
   - 当前做了双重校验（resolver + PathGuard），性能上多一次 stat
   - **结论**：安全优先，性能开销可忽略（每张图一次）

3. **preload.ts 仅用通道白名单 regex，而非更强校验**
   - 更强：在 preload 也做 Zod 预校验
   - **结论**：preload sandbox 下不能引入 zod 库；主进程已校验，已足够

4. **测试覆盖率门槛 80% vs per-file 门槛**
   - vitest 不直接支持 per-file，手写脚本复杂
   - **结论**：M1.5 完成前统一提升，retrospective 承诺记入
