## 🔍 方案合理性复盘

### 👍 好的决策

1. **Pass 拆分（3a/3b/3c）**：避免一个 commit 3000+ 行难审；3a 落地 2 个 shader 即可上线价值
2. **共享 VAO**：ShaderRegistry + Pass 复用 GLContext 的 quad VAO，省 setup 时间
3. **AbortController 步间检查**：滑块高频拖动时，新 pipeline.run 能打断旧的未完成 run
4. **ShaderCompileError 判别字段**：编译错 / 链接错分开暴露，future log 聚类友好
5. **不引第三方库（regl / twgl / pixi）**：600 LOC 全自研，依赖零、可审 100%
6. **WebGL 2 only（Q3-A）**：不写 WebGL 1 fallback 路径，渲染进程目标明确

### 🤔 可能改进

1. **GLContext 的 `preserveDrawingBuffer: false`** 后来被 M2 改成 true（为了直方图 readPixels）——P3a 当时没预留该需求导致 M2 改 4 处文件
2. **Pipeline.run 没返回 readPixels hook**：M2 的直方图只能用 `canvasRef.current.getContext('webgl2')` 再取一次；可以在 PipelineStats 里直接暴露 `sampleCanvas?(): ImageData`
3. **pipelineToSteps 当时签名 `(pipe, aspect)` 在 P3b 改成 `(pipe, BuildContext)`**：早点想到 BuildContext 扩展点能省一次重构
4. **registryRef 冗余**：useWebGLPreview 里有未读取的 ref，code-review M-1 指出，P3b-2 才删掉
