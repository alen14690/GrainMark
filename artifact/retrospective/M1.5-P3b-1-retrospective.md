## ✅ 达成项

### 🎨 7 个新 shader

| Shader | 文件 | Uniform | 关键算法 |
|---|---|---|---|
| whiteBalance | `shaders/whiteBalance.ts` | temp / tint | R/B 轴偏移（±30% 强度） |
| hsl | `shaders/hsl.ts` | 24 float（8 通道 × h/s/l） | Gaussian hue 加权（σ=30°）· 导出 HSL_CHANNELS 常量 |

| halation | `shaders/halation.ts` | amount / threshold / radius + u_texelSize | 红偏高光溢光，单 pass 9-tap 径向模糊 |

### 📐 Pipeline 集成

新的 Lightroom 9-step 顺序（每步附 `isIdentity()` 快速跳过）：

```
WB → Tone → Curves → HSL → ColorGrading → Adjustments → Halation → Grain → Vignette
```

- `pipelineToSteps(pipe, resolution)` 重写：每个 step 若 isIdentity 则不入队
- `hasGpuUnsupportedChannels()` 从 10 行判断收窄到只剩 `{ lut }`
- 移除 `registryRef`（code-review M-1）

## 🔍 方案合理性复盘

### 👍 好的决策

1. **Lightroom 顺序是事实标准**：WB → Tone → Curves → HSL → ColorGrading 的顺序在 Adobe / Capture One / Darktable 都基本一致，用户心智兼容
2. **每 shader 附 normalize + isIdentity + source contract 测试**：3 类断言形成质检三角形，后续 shader 修改难以悄悄退化
3. **adjustments 融合 clarity/sat/vibrance 为一个 pass**：三者都是 color-space 操作，共享一次纹理采样，减少 2 次 ping-pong
4. **HSL_CHANNELS 作为契约常量导出**：单测 + UI 都消费它，避免「前 3 个 float 是 red 还是 orange」的歧义
5. **7 shader 一起落地而非每次 1 个**：它们共享 pipelineToSteps 签名和测试风格，批量做更高效

### 🤔 可能改进

1. **halation 单 pass 的模糊半径上限 ~20px**：radius 调到 50+ 时条带明显；但 UI 滑块默认 0..30 所以没暴露
2. **curves 的 Hermite 插值切向计算**：相邻点距离不等时偶有轻微凹凸；非关键但 CurvesEditor UI（M4）上线后可能被肉眼注意到
3. **colorGrading 的 luma 蒙版阈值硬编码**（0.3 / 0.7）：没暴露给 UI，未来 Pro 用户可能需要自定义
4. **grain 用 hash21（2D hash）**：跨帧不稳定，但我们每次渲染都重新投骰子，视觉上正好是「动态颗粒」效果 —— 意外的好处
