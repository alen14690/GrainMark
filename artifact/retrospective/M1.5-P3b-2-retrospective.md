## 🔍 方案合理性复盘

### 👍 好的决策

1. **WebGL 2 原生 TEXTURE_3D 而非 2D tile atlas**：硬件三线性插值免费 / 无 padding 浪费 / 半像素校正只要 2 行 shader
2. **shared/cubeParser 抽出**：唯一事实源，0 Node 依赖，主渲染双侧消费零拷贝
3. **CubeParseError 带判别 code**：LUT_MISSING_SIZE / LUT_BAD_SIZE / LUT_DATA_MISMATCH / LUT_TOO_MANY_LINES 便于 UI 针对性文案
4. **WeakMap<GLContext, LRU>**：context 语义严格；当 useState 的 GLContext 被替换时旧 LRU 自动可 GC
5. **8-bit 量化**：感知测试下与 float 无差；避免 OES_texture_float_linear 依赖
6. **README 作为活文档**：每个 Pass 都更新，里程碑表 + 质量基线实时同步

### 🤔 可能改进

1. **LUT size > 64 的支持**：CUBE_LIMITS 当前硬限 64³，一些高端 LUT（如 Blackmagic / Academy）用到 33³ OK，但 65 / 100³ 的商业 LUT 会被拒绝；可观察用户反馈后放宽
2. **LUT 纹理上传没有 progress 反馈**：40ms 可见闪烁；M4 给 Editor 加 "LUT 加载中" 小 spinner
3. **sampler3D 在老 iGPU 的 fallback 策略缺失**：目前 useLutTexture 失败时 needsCpuFallback=true，但 CPU 端并没有真的实现 LUT 应用（IPC 那头的 applyPipelineSharp 对 LUT 是 no-op）—— 这是 M3 之前必须补的
