## 🔍 方案合理性复盘（专家自我 Review）

### 👍 好的设计决策

1. **Design Tokens 作为唯一真相源**：`tokens.ts` + `tailwind.config.ts` 强一致；未来要换主题（light / film）只需换 token
2. **字体栈分层使用**：正文 Inter / 标题 Fraunces 衬线 / 数字 IBM Plex Mono — 给 EXIF 和评分带来"仪表感"，是产品气质的关键
3. **全局颗粒层 `GrainOverlay` 独立组件**：opacity 3%，不干扰图像；未来可在设置中关闭
4. **ScoreBar 提前建空壳**：Pass 2 渲染接入 `score=null` 状态做引导；Pass 4 接入真实数据时无需改 UI 结构 — 这是 **接口先行、数据后到**的好实践
5. **Sidebar 分组（工作台 / Studio）**：更符合专业工具的心智模型（日常 vs 进阶）
6. **批量 sed 替换 + 保留复杂页面手写**：Batch/Filters 等页面内容多，深度重写收益低；P2 追求"统一视觉语言"而非"完美重构"，这是合理的**二八分工**

### 🤔 可能的改进方向（未来 Pass 可重审）

1. **动效细节暂不足**：虽有 presets，但页面切换/模态出入的动效没实际应用；P3/P4 接入 Framer Motion 或 CSS animation 系统化使用
2. **衬线字体 Fraunces 尚未广泛使用**：当前只用于页面标题；P3 会在 FilmStrip / Curves 标签上加强
3. **Sidebar 的 NavItem 在 `NavLink` render props 内部调用 `ref: Icon`，类型稍散**：后续可提成独立组件提升可读性
4. **颗粒层在 HiDPI 屏表现待验证**：Pass 2 仅桌面测试，retina 下 SVG 像素化可能偏柔或偏硬；用户反馈后可加 `@media (resolution: 2dppx)` 调参
